* Framework

Grid base, made with scaling, alignment, and mobile in mind

This is a CSS framework (with a touch of javascript for the dirty bits - none yet) based on the idea of a 12 column grid layout, infinite scalablity (high DPI), dynamic height vertical alignment, and responsive design. It also incorporates a column padding and margin as you'd expect from any layout or design program.

How does it work?

(in a general sense)

Column or Row Variables

Here's an overview of the changes you can make to a column or row

  • Grid Size: large_1, small_3, medium_4
  • Screen size adjustments: large_only, large_hide, medium_only, medium_hide, small_only, small_hide
  • Push or Pull: large_push_3, medium_pull_2, small_pull_1
  • Content Padding: content_padding - add space to the inside of a column
  • Content Margin: content_margin - add space to the outside of a column
  • Vertical alignment: valign_top, valign_middle, valign_bottom
  • Text alignment: text_right, text_center, text_left
  • Row Gutters add_gutters, remove_gutters

Overview

The class examples can be applied to any column element or row.

Rows or their parent elements can be floated left or right (float_left or float_right) and given a grid width (always_1, small_2, medium_3, etc.). Floating a row allows inline text to wrap around row sections.

Using the class add_gutters will add the padding to the left and right of each row, if you choose not to add content_margin spacing to the columns. You can see this below on rows without a content_margin class - the columns will be touching, but the sides will have padding from the edge of the page.

Example Code

<section class="content_item">
     <div class="row">
         <div class="large_12 column text_center">
             <span class="content">
                 <h1>Grid layout</h1>
                 <p>With content padding</p>
             </span>
         </div>
     </div>
</section>

Explanation

Here is the code to generate a content item with a row and columns. Think of each "content item" as a chunk of content on the page. Each row contains columns. Each column is given a size and contains a piece of content. Each piece of content can have a set padding or margin.

The column container gets most of the adjustment classes, but the row can also handle them, the effects will trickle down to the sub columns.

One unique feature of this framework is the ability to vertically align columns with adjacent columns - even when pushing or pulling them to rearrange the column order. Just apply the valign_top, valign_bottom, or valign_middle class to the column.

Code specifics

// This is separate from the grid layout and allows for equal spacing or not (bumping class)
 <section class="content_item bumping">
     // Defines a relative block element
     <div class="row">
         // Defines a relative inline-block element with a full 12 column width
          <div class="large_12 column">
             // Wraps the column content with a container to add margin or padding to
              <span class="content">
                 <h1>Grid layout</h1>
                 <p>With content padding</p>
             </span>
         </div>
     </div>
</section>

(Just a note)
Normally columns do not have a background or border. This effect is achieved by the class gray_columns applied to the row element.
